# Level Editor 使用说明（重点：编辑器操作）

本文档基于当前项目里的 Level Editor 实际实现编写，面向关卡设计与调试使用。

## 1. 快速上手（3 分钟）

1. 进入关卡编辑场景后，先在顶部确认关卡名与网格尺寸（`Width/Height`）。
2. 鼠标滚轮（中键）:是zoom in/zoom out. 按住鼠标滚轮拖动画布移动。
2. 左键功能介绍：左侧工具栏默认是 **Brush 模式**：
   - 选择绘制子模式（点 / 线 / 面 / 边缘）。
   - 选择要放置的实体类型。
   - 鼠标左键放置。在非“点”模式下，可以长按并拖拽鼠标试试看。
3. 需要批量移动或删除现有内容时，切到 **Select 模式**：
   - 框选/线选/点选目标；
   - 拖拽移动；
   - `Left Shift`按住可以多选
   - `Delete` 删除选中。
4. 右键功能介绍：永远是擦除模式，擦除范围和目前选择的工具相关。
  - 同样支持长按并拖拽鼠标试试看。
4. 完成编辑后点击顶部 **Save** 保存到 `StreamingAssets/Levels/<关卡名>.json`。
5. 可用 **Validate** 进入试玩验证；可用 **Solve** 让求解器尝试自动求解。

## 2. 界面区域说明

### 顶部 HUD

- 关卡名显示：未命名时显示 `（未命名）`。
- 网格宽高输入：范围 `1~50`，修改后立即生效。
- 常用按钮：
  - `Save`：保存当前关卡。
  - `Clear`：清空关卡实体（保留关卡名与 Metadata）。
  - `Validate`：进入/退出试玩。
  - `Solve`：后台求解并展示进度。
  - `Metadata`：打开元数据面板。
  - `Exit`：退出编辑场景（支持保存后退出 / 放弃后退出）。

### 左侧 Palette

- 大模式切换：
  - `Brush`（绘制）
  - `Select`（选择/移动）
- 子模式（两种大模式都支持）：
  - `点`（Point）
  - `线`（Line）
  - `面`（RectFill）
  - `边缘`（RectEdge）
- 实体列表：
  - 仅在 **Brush 模式**显示；
  - 按「基础物体 / 变种物体 / 装饰物体」分组。
- `Center` 按钮：
  - 将当前所有实体整体平移到画布中心附近。

## 3. Brush 模式（放置/擦除）

## 操作方式

- 左键：放置实体。
- 右键：移除实体。
- 在 `线/面/边缘` 模式下支持拖拽预览，松开鼠标后批量提交。

## 放置规则

- 同一格子同类型实体不会重复放置。
- 不可重叠实体会覆盖该格子内已有的不可重叠实体。
- 带 `OverlappableModel` 的实体可与其他实体共存。
- 文本实体（Text）会做占格冲突检查；若 footprint 冲突，则放置失败。

## 3.1 Text类实体
有时为了说明事情，可以在关卡中放置文字。
Text是一类特殊的实体：放置的实例(instance)在Select模式下被选中时，左侧UI会显示其可调整参数
+ 内容
+ 大小

## 4. Select 模式

Select 模式用于已有内容的编辑：选中、多选、移动、删除、右键擦除。

## 选择逻辑

- 在已选中目标上按下并拖拽：进入移动。
- 在未选中区域按下并拖拽：进入框选/线选/区域选。
- `Shift`：多选（追加到已有选择）。
- Point 子模式下，点击对象后可直接进入拖动语义。

## 移动逻辑

- 拖拽时会显示偏移（MoveOffset），松开后提交移动。
- 提交前会检查：
  - 是否越界；
  - 文本实体 footprint 是否越界或冲突；
  - 与未选中实体是否冲突；
  - 选中集合内部是否相互冲突。
- 任一检查失败则本次移动取消。

## 删除逻辑

- `Delete`：删除当前选中全部内容（支持撤销）。
- 右键：
  - Point：单点擦除；
  - 线/面/边缘：拖拽擦除并预览，松开后批量删除。

## 5. 撤销（Undo）

- 快捷键：`Ctrl + Z`。
- 支持撤销的典型操作：
  - 放置/擦除（含拖拽批量操作）；
  - Select 模式下的移动；
  - 网格尺寸修改；
  - Center 操作；
  - Clear（通过组合命令回退）。
- 当前仅实现 Undo 栈，未实现 Redo。

## 6. Validate 试玩验证

- 点击 `Validate` 进入试玩模式：
  - 编辑器对象和编辑 UI 会隐藏；
  - 当前关卡实体以可游玩逻辑重新生成；
  - 相机会自动适配关卡区域；
  - 播放当前关卡 BGM。
- 退出方式：
  - `Esc` 手动退出；
  - 或试玩通关后自动退出。
- 通关后会将关卡标记为 **可通关**（Metadata 的 `IsSolvable`）。

## 7. Solve 自动求解

- 点击 `Solve` 后会后台运行求解器，并显示进度。
- 成功时：
  - 自动标记关卡可通关；
  - 自动进入 Validate；
  - 自动回放求解动作。
- 当前求解器支持：
  - Type `0` Player
  - Type `1` Block
  - Type `2` Box
  - Type `3` Endpoint
  - `IsPureDecoration=true` 的纯装饰物
- 若关卡含不支持实体类型，求解会提示并终止。

## 8. Metadata 面板

通过顶部 `Metadata` 打开。

- 可编辑内容：
  - 关卡文件名（重命名，会同步更新 campaign 引用）
  - Display Name
  - Tags（添加/删除，支持已有标签下拉）
  - Difficulty（0~5，0.5 步进）
  - Comment
  - BGM 路径
  - 通关音效路径（全局音效设置）
- `Save & Close`：
  - 仅将 Metadata 写回关卡文件（不会覆盖实体布局）。
- `Discard & Close`：
  - 放弃本次面板内改动并关闭。

## 9. 保存/加载与文件位置

- 保存目录：`StreamingAssets/Levels/`
- 文件名：`<LevelName>.json`
- 建议流程：
  1. 先设置关卡名；
  2. 完成编辑后点击 `Save`；
  3. 再做 Validate / Solve；
  4. 需要时补充 Metadata 并 `Save & Close`。

## 10. 常见问题

- 不能放置/拖动：
  - 检查是否鼠标压在 UI 上；
  - 检查是否在 Validate 模式中。
- 移动不生效：
  - 目标可能越界或发生碰撞冲突（尤其是文本 footprint）。
- Metadata 仅保存失败：
  - 该关卡文件可能尚未创建，请先执行一次 `Save`。
- 网格改小后对象看起来“消失”：
  - 可能落在新边界外，建议先 `Center` 再调整尺寸。
