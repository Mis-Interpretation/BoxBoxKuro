建议的落地方案
分两步走，按收益排优先级：

第一步（高收益、低成本）：给全局 Model / 有状态的 Model 加接口分级 + DI

EditorStateModel → IEditorStateReader / IEditorStateWriter
TagRegistry → ITagRegistryReader / ITagRegistryWriter
这些用 Zenject 绑定即可，替换掉所有 FindAnyObjectByType
第二步（视项目规模决定）：每实体 Model

如果你的实体类型未来不会显著增长，现在的 MonoBehaviour 组合 + 约定已经够用。PositionModel 的字段改成属性、加个 set 里的 internal 或 [HideInInspector] 就行，不需要大改
如果你打算做更复杂的实体系统（比如动态添加能力、运行时组合），再考虑对每实体引入 DI
简单说：你的方向是对的，但不需要一步到位把所有 Model 都 DI 化。全局状态先上，每实体的量力而行。
